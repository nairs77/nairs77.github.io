How to use correctly 'self' keyword in Swift

2017-06-21
swift

원본 - How to use correctly ‘self’ keyword in Swift - Dmitri Pavlutin Blog

Swift에서 ‘self’는 인스턴스 자신을 가리키는 아주 특별한 property이다. Swift에서 ‘self’는 인스턴스 자신을 가리키는 아주 특별한 property이다. 주로 class나 structure 또는 enumeration의 초기화(생성자)나 method 호출에 사용한다. 명확하고 간결함을 추구하는 것을 모토로 삼기에는 아주 훌륭한 전략이다. 우선 대부분의 경우에 아주 효율적이고, 코드의 가독성을 높이는데 도와주나, 코드가 아주 짧아진다.

self에 이 모토를 적용하면, 아래와 같은 2가지의 모순이 발생한다.

명확성 전략을 적용하고 항상 인스턴스 속성 인 self.property에 액세스하려면 항상 자체를 유지해야합니까?
이 물음에 대답으로, Swift는 이 모든 것을 허용하고 심지어 self 키워드를 할 수 있다면 생략하는 것을 권장한다. self를 반드시 사용해야 할 곳과 선택적으로 사용할 것에 대해서 다루고자 한다.

Access instance properties and methods

init(winSpeed: chanceOfRain:)과 isDayForWalk 내부의 self는 Weather 구조체 instance를 기리킨다. Weather구조체의 self.windSpeed와 self.chanceOfRain를 접근(getter, setting)할 수 있는 것이다.

구조체는 전혀 문제가 없다.

하지만, 매번 self property에 접근하는 것은 좀 과한 것 같다. 구조체의 생성자와 메소드도 충분한 context를 제공한다. 모든 것들은 구조체 내부에서 일어나고 있다. self를 제공해도 될까요?
